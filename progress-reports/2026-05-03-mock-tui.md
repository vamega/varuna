# Mock TUI Client

## What Changed

- Added a `varuna-tui` binary scaffold with a lazygit-style mocked torrent dashboard.
- Added a reusable `varuna_tui` module under `src/tui/` with separate mock data, state, rendering, and terminal entry-point code.
- Added focused `zig build build-tui`, `zig build run-tui`, and `zig build test-tui` steps so the TUI can be built and exercised independently from daemon dependencies.
- Reworked the interactive client to run through ZigZag v0.1.3's `Program` model/update/view runtime.
- Added mock modal interaction for adding torrents, multi-mark selection styling, and pane-local navigation for filters and detail tabs.

## What Was Learned

- ZigZag v0.1.3 works with Zig 0.15.2 for this mock client, even though the package metadata still reports version `0.1.2`.
- The worktree's general `zig build test` path depends on system `sqlite3`; the focused TUI target avoids that daemon dependency.
- Observing the interactive client in tmux exposed terminal-specific issues that snapshots missed: ANSI escape bytes were counted as columns, raw mode disabled newline carriage returns, and full-height frames scrolled the pane.
- Moving to ZigZag removed the hand-written raw-mode terminal loop and gives us its `start()` / `tick()` path for later integration with daemon-style polling.
- Filter navigation can be modeled entirely in `AppState`: selecting the left-pane "Seeding" filter now narrows the mocked torrent table and keeps the selected torrent visible.

## Remaining Issues

- The TUI uses mocked data only. The next integration step is a real RPC/HTTP client feeding the same `AppState` shape or a thin snapshot model.
- The renderer still builds a custom lazygit-style view string; future work can replace more of that with ZigZag layout/table components if those APIs fit the desired density.
- The add-torrent modal intentionally discards input on submit for now; it proves the modal/key flow without committing to an RPC shape.
- Repo-wide tests were not completed in this environment because `sqlite3` was unavailable to Zig's system-library lookup.
