# 2026-05-01 AArch64 SHA1 asm compile fix

## What changed and why

- Fixed the AArch64 SHA1 inline assembly in `src/crypto/sha1.zig` so Zig/LLVM print operands with the correct AArch64 modifiers.
- Changed SHA1C/SHA1P/SHA1M to emit `Qd, Sn, Vm.4S` operands instead of vector-form operands that LLVM rejected after the first syntax issue was fixed.
- Made RPC form parameter decoding safe for tests that pass immutable request strings by copying request path/body at the API handler boundary and avoiding in-place URL decoding when the value has no encoded bytes.

## What was learned

- The apparent BoringSSL failure was a Varuna Zig inline-assembly failure on a final test link command that happened to include `libboringssl-*.a`.
- BoringSSL's own build model expects gas-compatible assembly on non-Windows AArch64; this was not a GNU-vs-LLVM assembler incompatibility in BoringSSL.
- Once the compile blocker was fixed, `test-api` exposed the existing unsafe `@constCast` in RPC parameter decoding.

## Remaining issues or follow-up

- `zig build test` still has unrelated runtime failures in peer policy, smart-ban, multi-source, and protocol tests. The full run reached `1792/1808 tests passed` before the stuck protocol test process was terminated.
