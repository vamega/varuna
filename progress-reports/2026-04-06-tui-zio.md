# TUI: Terminal UI for varuna using zigzag, zio, and dusty

## What was done

Rewrote `varuna-tui` to properly integrate zio as the event loop and dusty as
the HTTP client library.  Removed all stubs -- every feature now works
end-to-end.  Fixed memory management (no leaks from polling).  Added auth
support (login dialog) and preference editing.  Created tmux-based integration
tests.

### Architecture

- **zigzag** drives the TUI rendering using the Elm Architecture pattern
  (Model-Update-View).  Uses `start()` + `tick()` for manual loop control so
  it doesn't block the zio runtime.
- **zio** runtime runs with 2 executors: executor 0 (main thread, unused for
  zio) and executor 1 (worker thread running the event loop).  The poller task
  runs as a zio fiber on the worker executor.
- **dusty** HTTP client makes all API requests inside the zio fiber.  No
  threads are spawned for I/O -- dusty uses zio's async networking internally.
- **Thread-safe ResultQueue/ActionQueue** (using `std.Thread.Mutex`) bridges
  the zio poller and the zigzag main thread.  The poller pushes API results;
  the zigzag tick handler drains them.
- **Data arena** (`std.heap.ArenaAllocator`) holds current API data.  On each
  torrent list update, the arena is reset and new data is deep-copied in.
  No growing memory usage over time.

### Features implemented

**Main view:**
- Torrent list with columns: status icon, name, size, progress bar + %, download
  speed, upload speed, seed count, peer count
- Scroll support with j/k and arrow keys, Home/End
- Selected row highlighting with background color
- Empty state message

**Status bar:**
- Global download/upload speeds, torrent count, DHT node count
- Connection status indicator

**Key bindings bar:**
- Context-sensitive hints that change per view mode

**Modal dialogs:**
- Add torrent (file path or magnet URI, Tab to switch mode) -- calls API
- Confirm delete (with toggle for deleting files) -- calls API
- Preferences view (navigable, editable) -- fetches/sends via API
- Login dialog for auth-required daemons -- calls login API

**Detail view:**
- Torrent header (name, status, progress, speeds, seeds/peers, size, ratio)
- Three tabs switchable with Tab: Files, Trackers, Info
- Files tab shows index, name, size, completion %, priority
- Trackers tab shows URL and status
- Info tab shows hash, save path, pieces, connections, dates, category, tags
- Data fetched from API when entering detail view or switching tabs

**Actions (all wired to API):**
- Pause/resume: detects current state and sends opposite action
- Add torrent: file path (multipart upload) or magnet URI
- Delete torrent: with optional file deletion
- Set preferences: navigable list, edit individual fields

**Filter:**
- Case-insensitive name substring filter on the torrent list

**Auth:**
- Login dialog with username/password fields
- SID cookie management for subsequent requests
- 401/403 detection triggers login prompt

**Connection handling:**
- Disconnection overlay with retry information
- Automatic reconnection every 3 seconds on failure
- Auth required detection with graceful login flow

**HTTP API client (dusty):**
- All requests go through dusty's `Client.fetch()` inside zio fibers
- Cookie-based session (SID) support
- JSON response parsing via `std.json`
- Endpoints: torrents/info, transfer/info, torrents/properties, torrents/files,
  torrents/trackers, app/preferences, app/setPreferences, torrents/add,
  torrents/delete, torrents/pause, torrents/resume, auth/login

**CLI:**
- `--host HOST` and `--port PORT` options (default: 127.0.0.1:8080)
- `--help` flag with full usage documentation

## What was learned

- zio's `Runtime.spawn()` uses round-robin executor assignment starting at
  index 0.  Executor 0 is the "main" executor which only runs if the main
  thread calls `executor.run()`.  When the main thread is occupied (e.g. with
  a TUI loop), spawned tasks assigned to executor 0 will never execute.  Fix:
  set `rt.next_executor_index` to 1 so all spawns go to worker executors.

- dusty's `Headers` type is `std.HashMapUnmanaged` (Zig 0.15 unmanaged API).
  Initialize with `.{}`, pass allocator to `put()` and `deinit()`.

- dusty's HTTP `Method` enum uses lowercase names (`.get`, `.post`), matching
  llhttp's conventions.

- zigzag's `Program.start()` + `tick()` works well for custom event loop
  integration.  The `everyMs()` command sets up periodic ticks that drive
  result draining from the shared queue.

- zio's DNS resolver dispatches `getaddrinfo` to a thread pool via
  `blockInPlace`, which works transparently for IP addresses like 127.0.0.1.

- Zig 0.15 does not allow declarations between struct fields.  Named enums
  used as field types must be defined outside the struct.

- For cross-thread communication between a zio coroutine and a non-zio thread,
  `std.Thread.Mutex` is safer than `zio.Mutex` (which requires coroutine
  context for blocking operations).

### TorrentState enum (2026-04-08)

Replaced raw `[]const u8` state strings with a proper `TorrentState` enum in
`api.zig`.  Previously, torrent state was stored as a raw string (e.g.,
`"downloading"`, `"pausedDL"`) and compared throughout the codebase with
`std.mem.eql(u8, ...)`.  This was fragile -- typos in comparisons would silently
fail, and the compiler could not enforce exhaustive handling.

Changes:
- `api.zig`: Added `TorrentState` enum with 17 variants covering all qBittorrent
  API states.  Uses `std.StaticStringMap` for O(1) string-to-enum conversion in
  `fromString()`.  Methods: `displayString()` for human-readable labels,
  `statusSymbol()` for the list view icon, `statusColor()` returning `zz.Color`.
- `api.zig`: Changed `TorrentInfo.state` field from `[]const u8` to
  `TorrentState`.  JSON parsing calls `TorrentState.fromString()` during
  extraction.
- `views.zig`: Removed `statusSymbol()` and `statusColor()` free functions
  (moved to enum methods).  List rendering calls `t.state.statusSymbol()` and
  `t.state.statusColor()`.  Detail view uses `torrent.state.displayString()`.
- `main.zig`: Pause/resume logic uses `t.state == .pausedDL or t.state ==
  .pausedUP` instead of string comparison.  Removed `arena.dupe` for the state
  field since enums are value types.
